Problem

传统文献检索的三大痛点

🔍 搜索费时

数据库分散，检索词难优化，往往找不到真正相关的论文。

📚 筛选困难

结果海量，读完摘要也无法判断论文质量高低。

🤯 阅读门槛

英文论文结构复杂，不知道从哪里开始读、重点在哪里。

大家有没有这样的经历？写论文前要检索文献，在 Google Scholar、PubMed、arXiv 之间来回切换，检索词反复调整，找了半天还是漏掉了重要的论文。找到论文之后，摘要看完了也不知道这篇文章到底值不值得细读。读英文论文更是头疼，不知道先看哪部分、哪些是重点。

Search Papers 解决这三个痛点。首先，它集成了四大主流数据库——arXiv、CrossRef、Scopus 和 PubMed，一次搜索就能覆盖绝大部分学术资源。其次，它接入了 DeepSeek AI，可以帮我们自动优化检索词，生成结构化的分析报告。第三，支持引用链深度追踪，可以从一篇论文出发，追踪它的引用和被引用关系，找到这个领域的奠基工作和最新进展。最后，每篇论文都能生成专属的阅读指南，告诉我们应该先读什么、后读什么、哪些是重点图表。

Workflow

三步搞定文献检索

① 搜索

输入中文研究主题
AI 自动优化为英文查询
返回相关性排序的论文列表

② AI 分析

自动生成结构化报告
Top 5 推荐 · 分类列表
研究趋势 · 进一步建议

③ 细读

detail 命令查看单篇
AI 深度解读 · 阅读指南
图形摘要一键下载

整个工作流非常简洁，分三步。第一步搜索：输入中文研究主题，比如"大语言模型推理优化"，AI 会自动把这个主题优化成英文检索词，搜索四大数据库。第二步 AI 分析：搜索完成后自动提示，可以生成一份结构化报告，包含推荐论文 Top 5、按子方向分组的论文列表、研究趋势分析。第三步细读：用 detail 命令查看单篇论文，AI 会给出深度解读和专属阅读指南，还可以下载图形摘要。

Commands

常用命令一览

search <主题>

搜索研究主题（直接输入主题也支持）

depth <1-5>

设置搜索深度（1=单层，2+=追踪引用链）

analyze

对当前搜索结果进行 AI 分析

detail <编号>

细读单篇，含专属阅读指南

ask <问题>

就当前主题自由提问，AI 回答

guide

查看通用阅读框架（10秒定去留）

export [文件名]

导出搜索结果为 Markdown

save [文件名]

保存 AI 分析报告

这是常用命令一览。最核心的是 search、analyze、detail 三个。search 搜索主题，analyze 分析结果，detail 细读单篇。depth 命令可以设置搜索深度，数字越大追踪得越深，但也会花更多时间。ask 命令可以在当前主题下自由提问。guide 会展示一个通用的论文阅读框架，教大家 10 秒判断一篇论文值不值得读。还有 export 和 save 命令可以导出和保存结果。

Detail Report

每篇论文的专属阅读指南

📌 本文填补的研究空白

前人没做什么，本文做了什么

📖 推荐阅读顺序

先读哪个部分及原因

📊 核心图表

先看哪张图/哪个实验

⚠️ 潜在局限性

基于领域知识的客观批评

🏷️ 一句话标签

[年份]作者_核心结论_缺陷

🔗 找同类文献

祖师爷（奠基工作）和后浪（后续改进）

detail 命令生成的阅读指南非常实用。它会告诉我们六个方面的信息：第一，这篇论文填补了什么研究空白；第二，推荐的阅读顺序——比如应该先读 abstract 还是先读 introduction；第三，核心图表是哪张；第四，潜在的局限性是什么；第五，一句话标签方便快速记忆；第六，同类文献推荐，包括这个领域的奠基工作和后续的改进工作。这样我们读一篇论文，就能建立起对这个领域的全局认知。

优先使用 doi.org/10.xxxx

🔍 Google Scholar 保底

无 DOI 的论文使用

链接策略也是一个贴心的设计。工具会优先使用 DOI 直链，在校园网环境下可以直接跳转到出版商页面阅读全文。没有 DOI 的论文会用 Google Scholar 标题搜索作为保底。这样大家拿到链接后，基本都能顺利访问到论文。

Quick Start

5 分钟快速上手

Step 1 · 安装依赖

pip install -r requirements.txt

Step 2 · 配置 API Key

cp .env.example .env
# 填入 DEEPSEEK_API_KEY

Step 3 · 启动工具

python app.py

Step 4 · 开始检索

输入 research 主题，或直接用 search 命令

快速开始只需要四步。第一步安装依赖，requirements.txt 里已经列出了所有需要的包。第二步配置 API Key，把 .env.example 复制成 .env，然后填入你的 DeepSeek API key。第三步运行 python app.py 启动工具。第四步就可以开始检索了。输入研究主题，比如"transformer attention mechanism"，工具会自动优化检索词并搜索四大数据库。
